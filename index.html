XR_GAIT is a Unity-based mixed reality prototype that reproduces controlled gait tasks inside a laboratory environment, combined with post-acquisition movement review via Vicon motion capture.

Mixed reality tasks

Two controlled environments

Each scenario is designed to elicit specific gait-related behaviours while remaining compatible with simultaneous Vicon motion capture recording.

01 — Farm

Farm scenario

A linear corridor designed to reproduce controlled walking conditions with timed obstacles and interaction points.

02 — Park

Park scenario

An open environment with a central fountain and several interaction areas, designed for route-following and object manipulation tasks.

Technical pipeline

From capture to analysis

The prototype integrates real-time execution in the headset with a post-acquisition pipeline that reconstructs movement and generates structured event data.

01

Unity MR application

Task scenarios deployed to Meta Quest 3 via Android build.

02

Meta Quest 3 execution

Participant performs the task while wearing the headset in the lab.

03

Vicon motion capture recording

Simultaneous full-body marker recording during task execution.

04

Vicon Nexus reconstruction

Marker trajectories are labelled and reconstructed in Nexus.

05

Blender FBX conversion

Reconstructed skeleton exported as FBX for Unity import.

06

Unity playback

FBX animation replayed inside the original task scene.

07

CSV event output

Event colliders register task points and output event names, timestamps, positions and estimated forward speed.
